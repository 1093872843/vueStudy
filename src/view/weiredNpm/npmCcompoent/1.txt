创建一个npm 模块步骤
    1.你需要一个npm账号，去官网注册
    2.打开一个新的文件夹，执行 npm init,
    3.接下来会出现一堆提示让你填写内容
        name：填写你这个包的名字，默认是你这个文件夹的名字。不过这里要着重说一下，
              最好先去npm上找一下有没有同名的包。最好的测试方式就是，在命令行里面输入npm install 你要取的名字，如果报错，那么很好，npm上没有跟你同名的包，你可以放心大胆地把包发布出去。如果成功下载下来了。。。那么很不幸，改名字吧。。。
        version：你这个包的版本，默认是1.0.0
        description：这个描述你的包是干嘛用的，让别人知道直观知道这个包的作用
        entry point：入口文件，默认是Index.js，你也可以自己填写你自己的文件名。
        test command：测试命令，这个直接回车就好了，因为目前还不需要这个。
        git repository：这个是git仓库地址，如果你的包是先放到github上或者其他git仓库里，这时候你的文件夹里面会存在一个隐藏的.git目录，npm会读到这个目录作为这一项的默认值。如果没有的话，直接回车继续。
        keyword：这个是一个重点，这个关系到有多少人会搜到你的npm包。尽量使用贴切的关键字作为这个包的索引。我写的是 fall in love。
        author：写你的账号或者你的github账号吧
        license：这个直接回车，开源文件来着。。。
    4.完成后，你的目录下会生成一个package.json文件，文件中就是你上述填写的内容，如果有依赖，记得添加依赖。
    5.上述操作我们不是写了一个entry point的值为index.js吗，这里需要在目录下手动新建一个index.js文件作为文件入口。因为我的
       代码都在lib中，所以我的index.js文件中我只写了
          import test from './test.vue'
          export default test;
       主要作用是导出我们的文件内容，我是照着其他npm模块写，当然index.js文件名也可以起其他名字。
    6.最后我们只要把这个npm包推上npm就行了，npm publish
    7.最后一点要注意的是，如果你修改了这个自定义npm模块，需要再次推送的时候，就得去package.json
      中修改版本号，不然会推送失败。
    8.到这一步的时候，其实你的这个文件夹和node_module下的模块已经一样了，直接把这个文件丢进去node_module文件夹下，
        然后在你的其他代码里使用import就可以直接使用了，不需要文件路径或者相对路径了。
